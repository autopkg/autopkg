AutoPkg

AutoPkg is a system for automatically preparing software for distribution to managed clients. Recipes allow you to specify a series of simple actions which combined together can perform complex tasks, similar to Automator workflows or Unix pipes.

See the wiki for more information.

Installation

To install AutoPkg, unpack the distribution and run

sudo ./install.sh

This will set up a packaging server and a launchd job so regular users can create packages.

Configuration

Set MUNKI_REPO in ~/Library/Preferences/com.googlecode.autopkg.plist if you want recipes to populate your munki repository.

defaults write com.googlecode.autopkg MUNKI_REPO /path/to/your/munki/repo

Usage

/Library/AutoPkg/autopkg /path/to/recipe.plist

Getting Recipes

No recipes are included with the base autopkg tools.

There are many useful recipes at http://github.com/autopkg/recipes.

Current recommendation (using Git):

    mkdir ~/Library/AutoPkg
    cd ~/Library/AutoPkg
    git clone http://github.com/autopkg/recipes Recipes

This clones the recipes repo into ~/Library/AutoPkg/Recipes, where automunkiimport will find them automatically.
